Trending Topic Collection Application

2013 - iEntry, Inc.

jQuery, CakePHP, MySQL, CentOS

Description

I was the primary developer for this application. This project is a modification of an existing tool that is used by 50+ staff and contracted authors. Its main function is to keep track of topics covered by the writers and to gather potential trends. Trends are collected from Yahoo, Bing, and Google in addition to trend related articles. A weight is calculated based on the frequency they are seen. The page orders the trends based on their weight; the more popular the trend, the better the target. Over time, if a trend is not seen on search engines, it will slowly drop to the bottom of the list as it is no longer relevant. Individuals can claim each trend, the claim is indicated to all other writers.

Screenshots
Results

The implementation of this project directly resulted in a long-term increase in page views for the blog. The back-end systems were optimized to accommodate an increase in the number of writers. The system is extensible and modifiable to include additional trend sources. Functions were built-in to maintain a small database footprint and clean old data resulting in faster look-ups and an overall more efficient application.

Author Payment System

2013 - iEntry, Inc.

jQuery, CakePHP, MySQL, CentOS

Description

I was the primary developer for this application. This is a modification of an existing system that replaces manual weekly reports done by hand for 50+ authors. The reporting system collects relevant data from the database article records and generates a personalized email for each author Each author needs to confirm his or her report via email or via the interface. These reports are saved into a database for moderation. Totals are also calculated to provide an overview of that weeks report. Previous reports can be accessed via the pagination functions at the top of the page.

Screenshots
Results

Before this system was implemented, reports were calculated and collected by hand and manually confirmed with each of the 50+ authors. With this system handling all report generation and confirmation, hours of manual weekly reports are reduced to a minimum. In addition, this system can scale to handle additional authors as needed without increasing workload.
